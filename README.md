# Shift Planner

## Описание проекта

**Shift Planner** — мобильное приложение для автоматизации составления рабочих смен в кафе, ресторанах и других заведениях.

Проект решает проблему ручного планирования графиков через Excel, таблицы и чаты.  
Приложение помогает собирать доступность сотрудников, задавать требования к сменам и автоматически формировать расписание на месяц.

---

## Цель приложения

Цель приложения — упростить процесс планирования смен и сделать его удобным как для менеджеров, так и для сотрудников.

Приложение позволяет:
- учитывать доступность сотрудников
- задавать требования по количеству сотрудников и должностям
- автоматически генерировать смены
- редактировать итоговое расписание вручную

---

## Роли пользователей

### Менеджер
- создаёт компанию  
- добавляет рабочие должности  
- задаёт правила для смен  
- приглашает сотрудников  
- генерирует расписание  
- редактирует и добавляет смены  

### Сотрудник
- регистрируется и присоединяется к компании по коду  
- заполняет доступность на месяц  
- указывает желаемое количество часов/смен  
- просматривает своё расписание  

---

## User Flow

### Flow менеджера
1. Регистрация  
2. Создание компании  
3. Добавление должностей  
4. Настройка требований к сменам  
5. Приглашение сотрудников  
6. Генерация расписания  
7. Ручная корректировка смен  

### Flow сотрудника
1. Регистрация  
2. Ввод invite-кода  
3. Заполнение доступности  
4. Указание предпочтений по сменам  
5. Просмотр своего расписания  

---

## Архитектура приложения

Приложение построено на **SwiftUI + Firebase**.

### Основные сущности
- **User** — пользователь с ролью manager / employee  
- **Company** — компания  
- **WorkPosition** — должность  
- **ShiftRequirement** — требования к сменам  
- **GeneratedShift** — смена  
- **MonthlyAvailability** — доступность  
- **AvailabilityStatus** — статус дня  

### Основные модули
- Authentication  
- Company Management  
- Requirements Setup  
- Availability Calendar  
- Schedule Generation  
- Manual Schedule Editing

## Запуск проекта

### Требования
- macOS  
- Xcode  
- Firebase project  

### 1. Открыть проект
```bash
open ShiftPlanner.xcodeproj
```
### 2. Подключить Firebase

1. Создать проект в Firebase Console  
2. Добавить iOS-приложение  
3. Скачать `GoogleService-Info.plist`  
4. Добавить файл в Xcode в корень проекта  

### 3. Установить зависимости

Firebase подключается через Swift Package Manager:

- File → Add Package Dependencies  
- Добавить Firebase SDK  
- Подключить:
  - FirebaseAuth  
  - FirebaseFirestore  
  - FirebaseCore  

### 4. Настроить signing

- Target → Signing & Capabilities  
- Выбрать свой Apple ID  
- Убедиться, что Bundle Identifier уникален  

### 5. Запуск

Рекомендуется запускать приложение через Xcode:

- на iOS Simulator  
- или на подключённом iPhone  

---

### Примечание по `.ipa`

В проекте присутствует `.ipa` файл, однако он собран без полноценного Apple Developer distribution, поэтому может работать нестабильно или не устанавливаться.

Для корректного тестирования рекомендуется запуск через Xcode.
